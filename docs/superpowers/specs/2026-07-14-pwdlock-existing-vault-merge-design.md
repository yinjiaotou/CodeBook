# Pwdlock 已有密码库导入与冲突保留设计

## 目标与范围

在 macOS 已解锁密码库中提供“导入加密文件”入口。导入 `.pwdlock` v1 后，将不存在的登录项加入当前库、跳过完全相同的记录，并把同 ID 不同内容保存为待处理冲突。导入不得静默覆盖本地数据。

首版不处理非空墓碑数组，也不接收外部文件中已经存在的冲突组。遇到这两类载荷时整次导入失败并保持当前库不变。跨端文件格式仍以 `shared/protocol/pwdlock-v1.md` 为准。

## 方案选择

采用独立的冲突组与冲突变体表。相比给导入记录生成新 ID，此方案不会破坏逻辑记录身份；相比创建第二个密码库，它能实现真正的事务合并。冲突数据与登录项处于同一 SQLCipher 数据库中，磁盘上保持加密。

## 数据模型

新增 `conflict_groups`：

- `id`：冲突组 UUID，主键；
- `record_id`：产生冲突的登录项 UUID；
- `title`：用于冲突列表展示；
- `created_at_ms`：UTC Unix 毫秒；
- `state`：首版固定为 `pending`。

新增 `conflict_variants`：

- `id`：变体 UUID，主键；
- `group_id`：所属冲突组；
- `kind`：`local` 或 `imported`；
- `source_vault_id`：来源密码库 UUID；
- `payload`：完整 `LoginItem` JSON；
- 每组恰好保存一个本地变体和一个导入变体。

本地变体的 `source_vault_id` 记录当前密码库 ID，导入变体记录 `.pwdlock` 载荷中的 `sourceVaultId`；两者不得混用。

数据库迁移使用 `CREATE TABLE IF NOT EXISTS`，不改变已有 `login_items` 表。SQLCipher 继续承担本地静态加密。

## 合并规则

导入必须在已解锁状态进行，并在认证解密及严格载荷校验完成后开启单个数据库事务：

1. 目标库没有该记录 ID：插入导入记录，计为“新增”。
2. 目标库存在且全部逻辑字段相同：不写入，计为“已存在”。比较包含 ID、标题、用户名、密码、网址、分类、备注、创建/更新时间、修订号与设备 ID；创建/更新时间按 `.pwdlock` 协议的 UTC Unix 毫秒精度比较，本地 `Date` 的亚毫秒差异不构成冲突。
3. 目标库存在但内容不同：本地 `login_items` 保持不变；新建待处理冲突组并保存本地与导入两个完整变体，计为“冲突”。
4. 相同导入文件再次导入时，不重复创建等价的待处理冲突组。“等价”严格定义为：记录 ID、来源密码库 ID、本地完整载荷和导入完整载荷均相同；若本地记录已变化，则允许生成新的冲突组。
5. 载荷包含非空墓碑或外部冲突组：整次导入失败。
6. 任一 SQL、解码或约束错误：事务回滚，新增记录与冲突均不保留。

## 冲突处理

主界面显示待处理冲突数量，并提供冲突中心。冲突详情按字段对比本地与导入版本；密码默认遮蔽，用户明确操作后才显示。

- 保留本地：保留当前 `login_items`，删除冲突组及变体。
- 使用导入：用导入变体更新 `login_items`，随后删除冲突组及变体。
- 手动合并：预填本地标题、用户名、密码、网址、分类和备注，允许用户选择或编辑这些业务字段；保存时沿用本地记录 ID、创建时间和当前设备 ID，修订号设为两个变体最大值加一，更新时间使用当前 UTC 时间，随后删除冲突。

每次冲突裁决在单个事务内完成。首版不保存旧密码的长期历史。

## 用户流程

已解锁密码库工具栏新增“导入加密文件”：

1. 选择 `.pwdlock` 文件；
2. 输入独立导出密码；
3. 显示确认文案“不会静默覆盖本地记录”；
4. 完成后显示“新增 / 已存在 / 待处理冲突”摘要；
5. 有冲突时可直接进入冲突中心，也可稍后处理。

认证失败统一显示“密码错误或文件损坏”。不得显示路径、SQL、堆栈或条目内容。文件读取继续使用同一描述符执行大小验证和限量读取。

## 组件边界

- `PwdlockArchive`：只负责文件认证、解密和严格载荷验证；不访问数据库。
- `LoginItemRepository`：负责事务合并、冲突查询和冲突裁决。
- `VaultSession`：要求已解锁，协调 archive 解码与 repository 合并。
- `VaultAppState`：维护导入摘要、冲突数量和通用错误状态。
- SwiftUI：文件选择、导出密码表单、摘要和冲突中心；不接触 Vault Key 或数据库句柄。

## 测试与验收

- 新记录新增、相同记录跳过、不同记录生成一个冲突组；
- 本地版本在冲突解决前保持生效；
- 重复导入不产生重复冲突；
- 混合新增与冲突在同一事务内提交；注入失败时全部回滚；
- 非空墓碑/外部冲突组安全拒绝；
- 保留本地、使用导入、手动合并均正确删除冲突并更新记录；
- 密码在冲突列表和对比页默认遮蔽；
- 错误导出密码、损坏文件和超限文件不会修改当前库；
- 完整 `swift test`、release 构建、DMG 重建与签名验证通过。
