# Pwdlock sync service

NestJS + PostgreSQL service for authenticated, end-to-end encrypted password-vault synchronization.

## Security boundary

- Account registration and authentication are conventional server operations; account passwords are stored with Argon2id hashes.
- This server never receives a vault master password or a decrypted Vault Key.
- `vaults.encryptedKeyEnvelope` is generated and encrypted by the client.
- `vault_changes.ciphertext` and `vault_changes.signature` are opaque client-generated bytes. The service authorizes the Vault owner and stores/returns them unchanged.
- A device key can be revoked with `DELETE /v1/devices/:deviceID`; revocation prevents future writes but does not erase already-signed history.
- Password item fields must never be added as database columns, API DTO properties, logs, metrics labels, or error messages.

## Local development

```sh
cp .env.example .env
docker compose up -d postgres
npm install
npm run start:dev
```

The API listens at `http://localhost:3000/v1`. Use `npm test`, `npm run lint`, and `npm run build` before deployment.

For the existing cloud PostgreSQL instance, keep PostgreSQL bound to loopback and use an SSH tunnel (for example local `127.0.0.1:15432` to remote `127.0.0.1:5432`). Put the resulting URL in an untracked `.env`; never expose port 5432 publicly.

`TYPEORM_SYNCHRONIZE` is disabled by default. Do not turn it on for a shared database: application accounts intentionally do not own schema objects. Apply reviewed SQL migrations instead.

## Deployment

Build the image with `docker build -t pwdlock-serve .`. Production must provide a unique 32-byte-or-longer `JWT_SECRET`, a managed `DATABASE_URL`, HTTPS termination, backups, and a deliberate database-migration process. Apply SQL files in `db/init/` for a fresh database and then `db/migrations/` in lexical order. Do not enable TypeORM schema synchronization in production.

For the complete migration, server deployment, systemd, port-opening, verification, and rollback procedure, see [DEPLOYMENT.md](./DEPLOYMENT.md).
